# Оптимизация облачных серверов

Этот проект симулирует и оптимизирует политики управления кластером облачных серверов для эффективной обработки нагрузки.

## Функции

- **Прогнозирование нагрузки**: Использует экспоненциально взвешенное скользящее среднее (EWMA) для краткосрочных прогнозов.
- **Упаковка бинов**: Реализует алгоритм First Fit Decreasing (FFD) для размещения нагрузки.
- **Локальная ребалансировка**: Оптимизирует использование хостов через лёгкую миграцию.
- **Оценка задержки**: Моделирует задержку на основе использования хостов.
- **Симуляция политик**: Сравнивает реактивную, FFD и гибридную политики.

## Установка

1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd cloud_server_optimization
   ```

2. Создайте и активируйте виртуальное окружение:
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # На Windows: venv\Scripts\activate
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

## Использование

Запустите симуляцию:
```bash
python -m src.main
```

Это симулирует три политики (reactive, ffd, hybrid) и выведет их метрики производительности.

## Метрики

- **Среднее использование CPU**: Процент использования CPU на хостах.
- **Среднее количество активных хостов**: Количество хостов, поддерживаемых активными.
- **P95 задержка**: 95-й процентиль задержки в миллисекундах.
- **Процент пропусков SLO**: Процент времени, когда задержка превышает 180 мс.
- **Миграции за шаг**: Среднее количество миграций нагрузки за временной шаг.

## Конфигурация

Измените `src/config.py`, чтобы настроить параметры кластера, такие как ёмкость CPU хоста, запасы безопасности и бюджеты миграции.
